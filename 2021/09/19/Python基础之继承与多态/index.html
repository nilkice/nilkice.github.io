继承和多态

  • 在OOP程序设计中，当我们定义一个class时，可以从某个现有的class继承，新的class称为子类，而被继承的class称为基类、父类或超类
  • 如，编写一个动物类Animal，拥有一个run方法
    • class Animal(object):
      • def run(self):
        • print(‘The Animal is running…’)
  • 编写Dog和Cat类，继承于Animal，但什么都不写
    • class Dog(Animal):
      • pass
    • class Cat(Animal):
      • pass
  • 对于Animal来说，Dog是它的子类。对于Dog说，Animal是它的父类
  • 若父类拥有一些方法，则继承时子类自动拥有
    • dog = Dog()
    • dog.run()
    • cat = Cat()
    • cat.run()
  • 子类也可以重写父类的方法，或增加自己的方法，如Dog类
    • class Dog(Animal):
      • def run(self):
        • print(‘The Dog is running…’)
      • def eat(self):
        • print(‘The Dog eating meat…’)
  • 若子类Dog重写了父类Animal的run()方法，调用run()方法时，执行子类的run()方法
    • dog.run()
  • 这种子类方法覆盖父类方法，称之为 多态
  • 判断dog的数据类型
    • print(isinstance(dog, Dog))
    • print(isinstance(dog, Animal))
  • 其结果均为true，表示dog不仅是Dog类型，还是Animal类型
  • 要理解多态，在定义一个函数
    • def run_twice(animal):
      • animal.run()
      • animal.run()
  • 当增加一个Animal的子类时，不需要对run_twice()做任何修改，就可以使用run_twice()方法，而且若子类重写了run()方法，就可以调用子类的run()方法，这就是多态的好处。
  • 这体现了著名的开闭原则
    • 对扩展开放：允许增加Animal子类
    • 对修改封闭：不需要修改依赖于Animal类型的函数方法
  • Python为动态语言，这意味着run_twice()函数中的参数animal，并不严格限制为Animal类，实际上只要传入的实例中有run()方法就行
    • class Robot(object):
      • def run(self):
        • print(‘Robot like a Animal running…’)
    • run_twice(Robot())
